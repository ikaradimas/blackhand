# BlackHand

**A neon-noir BitTorrent client.**

Cross-platform desktop torrent client for **Windows / macOS / Linux**, powered by [`librqbit`](https://github.com/ikatson/rqbit) and Tauri 2. Personal/hobby project; v0.1 functionally complete.

---

## Status

`v0.1.0` — feature-complete for personal use. Builds locally on all three desktop platforms; signing/notarization for redistribution is **not** wired up (see [Distribution](#distribution)).

## What it does

- Add torrents from **magnet links** or `.torrent` files (paste, drag-and-drop, browser click, "Open with…")
- Live progress, speeds, peers, ETA at 2 Hz
- Pause / resume / forget (keep files) / delete (remove files) — single-click actions, plus bulk Pause-all / Resume-all from header or tray
- **Per-file include/skip** in the detail panel — apply to skip parts of a multi-file torrent
- **Sequential download** by default — head pieces arrive first, so streaming-style playback works as soon as enough is buffered
- **Native completion notifications** — banner / toast when a torrent finishes
- **System tray** — left-click toggles the window, right-click for menu (Show / Pause-all / Resume-all / Quit)
- **Keyboard shortcuts** — ⌘N add, ⌘, settings, ⌘W close-to-tray
- **Persistence** — torrents resume across restarts via librqbit's fastresume
- **Settings** — listen port range, UPnP, DHT, global up/down KB/s caps. Bandwidth caps apply live; the rest take effect on next launch.

The aesthetic is **Neon Noir**: magenta + cyan on near-black, mono numerics with tabular figures, subtle scanline overlay, 5×5 pixel-art identicons per torrent. Designed to be readable for long sessions, not just to look cool — see `tasks/todo.md` §5 and `memories/2026-05-02-aesthetic-neon-noir.md` for the design system.

## Building from source

### Prerequisites

- **Node 20+** and **pnpm** (`npm i -g pnpm`)
- **Rust stable** (`rustup install stable`)
- Tauri 2 platform prereqs: see [tauri.app/start/prerequisites](https://v2.tauri.app/start/prerequisites/)
  - macOS: Xcode Command Line Tools
  - Windows: WebView2 (preinstalled on Win 11), Microsoft C++ Build Tools
  - Linux: `webkit2gtk-4.1`, `libssl-dev`, etc.

### Dev loop

```bash
pnpm install
pnpm tauri dev
```

First run takes ~1–2 min while `librqbit` and friends compile; incremental rebuilds are fast (~10–15 s).

> **Windows note:** if `pnpm install` fails inside `esbuild`'s postinstall with `"The system cannot find the path specified"`, run `pnpm install --ignore-scripts` instead. This is a pnpm 10 quirk — `.npmrc`'s `ignore-scripts=true` is read but not enforced for dependency *build* scripts on Windows. esbuild's postinstall is just a symlink convenience; the native binary still installs via the platform-specific optional dep, and esbuild's runtime locates it without it.

### Production build

```bash
pnpm tauri build           # release bundle for current platform
pnpm tauri build --debug   # debug bundle (faster, larger; useful for verifying packaging)
```

Outputs land under `src-tauri/target/{debug,release}/bundle/` — `.app` + `.dmg` on macOS, `.msi` + `.exe` on Windows, `.AppImage` + `.deb` on Linux.

### Type bindings

The Rust → TypeScript IPC bindings at `src/lib/bindings.ts` are generated by [`tauri-specta`](https://github.com/specta-rs/tauri-specta). They auto-regenerate on every `tauri dev` launch. To regenerate without launching:

```bash
pnpm gen-bindings
```

## Where things live

| What | Location (macOS) | Linux | Windows |
|---|---|---|---|
| Settings | `~/Library/Application Support/blackhand/settings.json` | `$XDG_DATA_HOME/blackhand/settings.json` | `%APPDATA%\blackhand\settings.json` |
| Resume data | `~/Library/Application Support/blackhand/session/` | `$XDG_DATA_HOME/blackhand/session/` | `%APPDATA%\blackhand\session\` |
| Logs | `~/Library/Logs/app.blackhand.client/blackhand.log` | `$XDG_STATE_HOME/app.blackhand.client/logs/` | `%LOCALAPPDATA%\app.blackhand.client\logs\` |
| Downloads (default) | `~/Downloads/BlackHand/` | `~/Downloads/BlackHand/` | `%USERPROFILE%\Downloads\BlackHand\` |

The About modal has an "Open log folder" button.

## Known limits

- **No MSE/PE wire encryption.** Networks that throttle plaintext BT may impact transfer rates.
- **No BitTorrent v2.** v1 / hybrid-v1 only. (Both `librqbit` constraints — see `memories/2026-05-02-engine-librqbit.md`.)
- **No tracker URL list / per-tracker re-announce in the UI yet.** librqbit's public API doesn't expose them cleanly. Deferred to v0.2.
- **No peer list panel** (the per-torrent peer *count* is shown). Deferred.
- **No RSS, no built-in search, no scheduler, no Tor/I2P** — explicitly out of scope for v0.1. See `tasks/todo.md` §7 for the v0.2 candidate list.
- **macOS deep-link / file-association testing in `tauri dev`**: LaunchServices only learns about `magnet://` and `.torrent` after a real `.app` bundle is registered. `pnpm tauri build --debug` once is enough to wire those up for testing.

## Distribution

Local builds work; **signed/notarized** distribution is not set up:

- **macOS:** `.app` and `.dmg` are produced unsigned. To distribute, you need an Apple Developer ID + `notarytool` workflow. Add cert details in `tauri.conf.json` under `bundle.macOS.signingIdentity` + `bundle.macOS.providerShortName`, plus `notarize` in the build command.
- **Windows:** `.msi` and `.exe` are produced unsigned. SmartScreen will warn end users until you sign with a code-signing certificate from a CA (~$80–200/yr).
- **Linux:** `.AppImage` and `.deb` are usable as-is; signing is optional.

## Architecture (brief)

- **Backend:** Rust + Tauri 2. `librqbit::Api` is wrapped as `Arc<Api>` and stored as Tauri managed state. Stats poll loop emits `torrents:snapshot` + `session:stats` at 2 Hz.
- **Frontend:** SvelteKit (SPA mode) + TypeScript. Svelte 5 rune-based stores in `.svelte.ts` files for session/torrents/ui. Native `<dialog>` element for modals.
- **IPC:** typed via `tauri-specta`. Single source of truth in `src/lib/bindings.ts` — never edit by hand.
- **Plugins:** `tauri-plugin-{opener, deep-link, single-instance, notification, log}`.
- **Design system:** CSS custom properties in `src/lib/design/tokens.css`, decorative effects in `effects.css`.

For deeper context, see the decision memos in `memories/`.
