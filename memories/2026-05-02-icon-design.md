# App icon: magenta "B" with cyan viewfinder brackets

**Date:** 2026-05-02
**Status:** Decided

## Decision
App icon is a Helvetica Neue 900 "B" in magenta (`#FF2A6D`) centered on a near-black canvas (`#0E0E16`), with four cyan (`#08F7FE`) corner brackets framing it like a scanner viewfinder. SVG source lives at `src-tauri/icons/app-icon.svg`; the 1024×1024 PNG export at `src-tauri/icons/app-icon.png` is the input to `pnpm tauri icon` for regenerating all platform sizes.

## Why
User asked for the cyan + magenta palette consistent with the boot wordmark. The viewfinder-bracket motif ties the icon to the same `[ ]` chrome used around the wordmark in `src/routes/+page.svelte` and gestures at "scanner / surveillance" — on-aesthetic for a torrent client.

## Alternatives considered
- **Block-pixel B composed of rectangles** (first attempt) — read as a closed rectangle with two square windows, not unmistakably a B. Rejected.
- **Custom SVG path B** — would render identically regardless of font availability, but more work to author and tweak. Not worth it when system Helvetica resolves cleanly via fontconfig.
- **Pure black background** — gets lost on dark OS docks/taskbars. `#0E0E16` adds just enough lift to separate from the surface.

## Consequences
- Mobile icons (`android/`, `ios/`) generated by `tauri icon` were deleted — desktop-only project.
- To re-generate after editing the SVG: `rsvg-convert -w 1024 -h 1024 src-tauri/icons/app-icon.svg -o src-tauri/icons/app-icon.png && pnpm tauri icon src-tauri/icons/app-icon.png`. Then delete the regenerated `android/` and `ios/` dirs.
- librsvg is now a development dependency (installed via `brew install librsvg`). Document in README when README gets written.
- Helvetica Neue is macOS-only; if anyone iterates on the SVG on Linux/Windows, the font fallback chain (Helvetica → Arial → sans-serif) will likely render slightly differently. Not a blocker for occasional updates.
